Block Ciphers

Modes of Operation

  • ECB (Electronic Codebook): Each block encrypted independently. Reveals patterns → insecure.
  • CBC (Cipher Block Chaining): Each block XORed with previous ciphertext before encryption. Errors propagate.
  • CTR (Counter Mode): Turns block cipher into stream cipher. Parallelisable, no error propagation.

Components

  • XOR: Combines plaintext with key bits.
  • Circular Shifts: Move bits around to increase diffusion.
  • S-Boxes: Non-linear substitution to provide confusion.

Worked Example (like Plover in sample test)

Encrypting a block (in binary):

  1. Key XOR: Plaintext ⊕ Key = Result A.
  2. Circular Shift Right: Shift Result A = Result B.
  3. Substitution: Replace bits using S-Box = Ciphertext.

This demonstrates diffusion (bit shuffling) and confusion (substitution), key concepts in block cipher security.

Practice Questions

  1. Which block cipher mode leaks patterns in the ciphertext?
    ECB.
  2. Which step in a block cipher provides confusion?
    S-Boxes (substitution).
  3. If a ciphertext block is corrupted, which mode propagates the error to the next block?
    CBC.
  4. Which block cipher mode can be parallelized for faster encryption?
    CTR (Counter Mode).
  5. What operation is typically used to combine plaintext with key bits?
    XOR.
  6. What is the main purpose of circular shifts in block ciphers?
    To increase diffusion by moving bits around.
  7. Name two key principles of block cipher security.
    Confusion and Diffusion.
  8. Which block cipher mode does not propagate errors if a block is corrupted?
    ECB or CTR.
  9. In CBC mode, what is XORed with the plaintext block before encryption?
    The previous ciphertext block (or IV for the first block).
  10. Why is ECB considered insecure for large amounts of data?
    Because identical plaintext blocks produce identical ciphertext blocks, revealing patterns.
  11. Which block cipher mode is most suitable when encryption and decryption need to be performed in parallel?
    CTR (Counter Mode).
